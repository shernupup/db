#1.2. 构建工程结构

启动kdevelop，创建一个目录，并用kdevelop打开这个目录生成一个工程

工程目录规划如下:
-db
   -contrib
         +libpoco
   +cmake
   -db
         +src
   -util
         +common
   +notes

每个目录下都会有个CMakeLists.txt文件
其中，libpoco存放解压后的poco源码包
cmake为一些cmake规则文件
db为工程主要核心代码区
util为一些工具类(common为一些通用工具，如字符串工具)
notes为工程推进过程中的笔记信息

#2. 从CMakeLists.txt开始

#2.1. 根目录下的CMakeLists.txt

#工程名称
project(db)
#cmake版本要求检查
cmake_minimum_required(VERSION 3.2.0)
# 设置根目录
SET(PROJECT_ROOT_PATH "${CMAKE_SOURCE_DIR}")
#是否不启用系统Poco，由于我们直接依赖源码包，因此设置为1
option(NOT_UNBUNDLED "not to use system poco library" 1)

#开启c++11,不直接使用add_definitions(-std=c++11)的原因是：
#编译c代码时会有警告:
#cc1: warning: command line option ‘-std=c++11’ is valid for C++/ObjC++ but not for C
if(CMAKE_COMPILER_IS_GNUCXX)
    set(CMAKE_CXX_FLAGS "-std=c++11 ${CMAKE_CXX_FLAGS}")
    message(STATUS "optional:-std=c++11")   
endif(CMAKE_COMPILER_IS_GNUCXX)

#打印当前根目录路径（绝对路径）
message("project root path is : " ${PROJECT_ROOT_PATH} )

#执行集成poco的相关cmake命令
include(cmake/find_poco.cmake)

#添加子目录
add_subdirectory(contrib)
add_subdirectory(util)
add_subdirectory(db)

以上注释比较详细，现在我们关注cmake/find_poco.cmake
需要注意的是，发布时，Poco源码包根目录下的CMakeLists.txt
中的POCO_STATIC选项可以开启，表示编译成静态链接库

以上是find_poco.cmake的关键片段，其实也就是指明了#include时需要搜索的路径

#2.2. 子目录下的CMakeLists.txt

以util/common下的CMakeList.txt为例来说明问题，其他CMakeLists.txt类似

aux_source_directory(. COMMON_UTIL_DIR)

add_library(CommonUtil STATIC ${COMMON_UTIL_DIR})

很简单的两句话，aux_source_directory表示源文件位于当前目录
,且赋值给COMMON_UTIL_DIR
add_library表示构建一个名为CommonUtil的链接库(STATIC表示静态库)
